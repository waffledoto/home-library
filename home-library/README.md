# ☕ Домашняя Библиотека

Веб-приложение для управления личной коллекцией книг с встроенным PDF-ридером. Разработано с использованием Next.js, TypeScript, PostgreSQL и Tailwind CSS. Файлы книг и обложек хранятся в **Vercel Blob** — облачном хранилище, которое работает с деплоями на Vercel.

## 🎨 Дизайн

Приложение выполнено в красивых кофейных тонах с градиентами и современным UI:
- **3 колонки** для управления статусом книг: "Не начатые", "Читаю", "Прочитанные"
- **Встроенный PDF-ридер** для чтения книг прямо на сайте
- **Адаптивный дизайн** для работы на любых устройствах
- **Красивые анимации** и плавные переходы

## ✨ Функциональность

### Управление книгами
- ➕ Добавление новых книг с обложкой, описанием и PDF-файлом
- ✏️ Изменение статуса книги (не начата / читаю / прочитана)
- 📊 Отслеживание прогресса чтения по страницам
- 💾 Сохранение прогресса чтения
- 🗑️ Удаление книг

### PDF-ридер
- 📖 Чтение PDF файлов прямо в браузере
- 📁 Возможность открыть локальный PDF-файл с компьютера
- 🔄 Автоматическое сохранение текущей страницы
- 📱 Удобный интерфейс для навигации по страницам

### Загрузка файлов
- 🖼️ Загрузка обложек книг (любые изображения) в Vercel Blob
- 📚 Загрузка PDF файлов в облачное хранилище Vercel Blob
- ☁️ Файлы надёжно хранятся в облаке и доступны после деплоя

## 🚀 Установка и запуск

### Требования
- Node.js 18+
- npm или yarn
- Аккаунт на [Vercel](https://vercel.com)
- Облачная база данных PostgreSQL (например, [Neon](https://neon.tech) или [Supabase](https://supabase.com))
- Vercel Blob хранилище (настраивается в панели Vercel)

### 1. Настройка Vercel Blob

1. Зайдите в панель управления Vercel → ваш проект
2. Перейдите во вкладку **Storage** → **Blob** → **Connect** или **Create**
3. Создайте Blob хранилище (бесплатно 1 GB)
4. Скопируйте токен **`BLOB_READ_WRITE_TOKEN`** из настроек проекта

### 2. Настройка PostgreSQL

1. Создайте базу данных на [Neon](https://neon.tech) / [Supabase](https://supabase.com) или другом провайдере
2. Скопируйте строку подключения (connection string) вида:
   ```
   postgresql://user:password@host:port/database?sslmode=require
   ```

### 3. Установка зависимостей
```bash
npm install
```

### 4. Настройка переменных окружения

Создайте файл `.env.local` (не коммитится в git):
```bash
cp .env.example .env.local
```

Заполните переменные:
```env
POSTGRES_URL=postgresql://user:password@host:port/database?sslmode=require
BLOB_READ_WRITE_TOKEN=vercel_blob_...
```

### 5. Запуск в режиме разработки
```bash
npm run dev
```

Приложение будет доступно по адресу: [http://localhost:3000](http://localhost:3000)

### 6. Деплой на Vercel

1. Запушите код в GitHub
2. Подключите репозиторий в Vercel
3. В настройках проекта на Vercel добавьте переменные окружения:
   - `POSTGRES_URL` — строка подключения к PostgreSQL
   - `BLOB_READ_WRITE_TOKEN` — токен из Blob хранилища
4. Деплой произойдёт автоматически

### Сборка для продакшена (локально)
```bash
npm run build
npm start
```

## 📖 Использование

### Добавление книги

1. Нажмите кнопку **"Добавить книгу"** в шапке
2. Заполните поля:
   - **Название книги** (обязательно)
   - **Автор** (обязательно)
   - **Описание** (необязательно)
   - **Обложка** (необязательно, изображение)
   - **Файл книги** (необязательно, PDF)
3. Нажмите **"Добавить книгу"**

### Чтение книги

1. Кликните на карточку книги
2. Откроется PDF-ридер
3. Если у книги есть PDF-файл, он загрузится автоматически
4. Можно открыть локальный PDF-файл кнопкой **"Открыть PDF"**

### Изменение статуса

1. В PDF-ридере нажмите на кнопку статуса
2. Выберите новый статус:
   - 📚 **Не начата** - книга ещё не начата
   - 📖 **Читаю** - книга читается
   - ✅ **Прочитана** - книга прочитана
3. Нажмите **"Сохранить"** для сохранения прогресса

## 🔌 API Эндпоинты

### Книги
- `GET /api/books` - Получить все книги
- `POST /api/books` - Создать новую книгу
- `GET /api/books/[id]` - Получить книгу по ID
- `PUT /api/books/[id]` - Обновить книгу
- `DELETE /api/books/[id]` - Удалить книгу

### Загрузка файлов
- `POST /api/upload` - Загрузить файл

## 📝 Примечания

- База данных PostgreSQL подключается через переменную `POSTGRES_URL`
- Файлы книг и обложек хранятся в **Vercel Blob** (облачное хранилище)
- При удалении книги файлы автоматически удаляются из Blob хранилища
- Vercel Blob бесплатно предоставляет 1 GB хранилища
- Для локальной разработки необходимы оба: PostgreSQL и Vercel Blob токен

## 🐛 Решение проблем

### Ошибка при запуске
Убедитесь, что все зависимости установлены и переменные окружения настроены:
```bash
npm install
```
Проверьте, что файл `.env.local` содержит `POSTGRES_URL` и `BLOB_READ_WRITE_TOKEN`.

### Ошибка подключения к базе данных
Проверьте строку подключения `POSTGRES_URL` в `.env.local`. Убедитесь, что база данных доступна и поддерживает SSL.

### Ошибка загрузки файлов (Blob)
Убедитесь, что `BLOB_READ_WRITE_TOKEN` корректно скопирован из настроек проекта Vercel.
